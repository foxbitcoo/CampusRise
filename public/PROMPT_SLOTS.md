# 输入槽位设计

## 设计原则

这个项目后续不建议一上来就让用户填很多信息，而是先尽量从图片里拿信息，再只向用户补问必要内容。

新版原则是：

1. 先看图，再问人
2. 先自动判断，再人工确认
3. 用户只传一张图时也要尽量工作
4. 学校名称必须和用户确认，因为它决定学校元素、校训、地标和校友参考
5. 知名校友和校训不要求用户输入，由模型在确认学校后检索并整理

这样做的原因：

- 更利于技能封装
- 更贴近真实用户上传习惯
- 更容易减少用户输入负担
- 更容易增强学校识别度和共鸣感
- 更容易把画面主题保持在积极、昂扬、向上的方向

## 输入分层

### 必填输入

- `人物肖像图片`
  - 用途：提供人物外形、气质、年龄层、构图基础
  - 说明：允许只上传一张图，不强制要求用户先准备正脸和侧脸

### 必须确认的信息

- `学校名称`
  - 用途：决定学校地标、校训、校史、校友参考和整体视觉识别
  - 说明：优先从图片中的校徽、校名、校服、建筑文字、场景信息里自动识别
  - 规则：即使从图片中识别到学校，也必须向用户确认
  - 推荐话术：
    - “我从图片里看到的学校线索像是【XX大学】，请确认这是你的学校吗？”
    - “我没能从图片里可靠识别学校，请告诉我学校名称，我再继续。”

### 建议补充槽位

- `英文名 / 签名名`
  - 用途：海报中的签名或题字
  - 不填时处理：不强行加入签名

- `专业 / 学院`
  - 用途：增强画面的专业气质和个人路径
  - 示例：计算机、法学、建筑、医学、新闻传播、金融、公共管理、艺术设计、自动化
  - 不填时处理：先走通用校园叙事

- `毕业年份`
  - 用途：控制海报落款、毕业季时间和画面文字
  - 示例：2026
  - 不填时处理：如果判断为刚毕业或即将毕业，读取当前时区年份作为默认毕业年份；如果判断为毕业多年校友，不编造年份，只表达“毕业多年”
  - 规则：一旦确定毕业年份，图中所有年份文字都必须一致，不能让模型自由生成错误年份

- `个人特质 / 希望呈现的气质`
  - 用途：让画面更像这个人，而不是只像学校
  - 示例：勇敢、自由、坚定、温柔、理性、热烈、独立、探索、创造、笃定、开阔
  - 不填时处理：从人物照片中提取气质线索

- `校园经历 / 社团经历`
  - 用途：补充更有个人记忆的校园内容
  - 示例：学生会、辩论队、合唱团、实验室项目、支教、创业比赛、毕业晚会、社团展演、图书馆自习、操场夜跑
  - 不填时处理：优先使用学校地标、校训和公共校园记忆

## 不要求用户填写的信息

- `知名校友`
  - 用户通常不会主动提供
  - 模型应在学校确认后自行检索
  - 只使用确认真实存在、知名度较高、形象正向的校友
  - 如果没有足够把握，不要编造

- `校训`
  - 模型应在学校确认后自行检索
  - 校训是学校认同感和视觉主题的重要来源
  - 如果检索不到可靠信息，不要编造

- `学校代表性视觉元素`
  - 例如北大的未名湖、清华的校门、学校主楼、图书馆、钟楼、校徽、校园轴线、代表性雕塑
  - 模型应在学校确认后自行检索并提炼
  - 优先使用能让用户快速识别“这就是我的学校”的元素

## 交互建议

技能在接收输入后，建议先判断信息完整度，再决定是否直接生成。

推荐规则：

1. 没有 `人物肖像图片` 时，不直接生成，先要求补图。
2. 有 `人物肖像图片` 时，先从图片判断人物年龄层，并粗分为：
   - `30 岁及以下`：优先判断为刚毕业或即将毕业的人
   - `30 岁以上`：优先判断为已经毕业比较久的校友
3. 有 `人物肖像图片` 时，优先尝试从图里识别学校：
   - 校徽
   - 校服文字
   - 建筑或门牌上的学校名
   - 毕业照现场中的横幅或标识
4. 无论是否识别到学校，都必须和用户确认学校名称。
5. 学校确认后，检索学校代表元素、校训、正向知名校友，再进入生成。
6. 如果缺少 `专业 / 学院` 或 `个人特质 / 校园经历`，可以生成，但应给用户一个友好的补充机会：
   - “学校我已经确认了。如果你没有特别说明毕业年份，我会默认按今年毕业处理。你也可以补充专业，比如计算机、法学、建筑；或者补充几个希望呈现的个人特质，比如勇敢、自由、坚定。没有也可以继续生成。”

## 单技能分流策略

不建议为“应届毕业版”和“校友回望版”分别做两个独立 skill。

更推荐的方式是：

- 只保留一个总 skill 名称
- skill 内部根据图片和少量澄清自动判断走哪个方向
- 不把内部版本名暴露给用户

### 对用户的话术

不要对用户说：

- “我会按校友回望版来做”
- “我会按应届毕业版来做”

推荐说：

- “我判断您更像是刚毕业或即将毕业的同学，我会重点做毕业仪式感和校园成长线。”
- “我判断您更像是已经毕业比较久的同学，我会重点做母校回望、时间沉淀和继续向上的力量感。”

### 内部分流规则

优先级建议如下：

1. 先看图片里的年龄层
   - 如果明显 `30 岁及以下`，内部走 `应届毕业版`
   - 如果明显 `30 岁以上`，内部走 `校友回望版`

2. 再看用户是否明确指定主题
   - 如果用户明确说“应届毕业”“毕业照”“毕业典礼”“学位服”等，内部走 `应届毕业版`
   - 如果用户明确说“校友”“毕业多年”“回望母校”“校史”“传承”等，内部走 `校友回望版`

3. 如果用户纠正，立即调整
   - 博士应届、延期毕业、返校拍毕业照等情况，允许改回应届毕业方向
   - 年轻校友、旧照片、纪念多年后的母校关系等情况，允许改成校友回望方向

## 学校语言规则

学校名称一旦确认，生成时使用以下硬规则：

1. `中国大陆高校`：统一使用简体中文
2. `香港 / 澳门 / 台湾高校`：使用当地常用中文写法；没有更明确要求时，优先使用繁体中文
3. `海外高校`：统一使用英文
4. 用户只说“清华大学”时，默认指北京的清华大学，使用简体中文“清华大学”
5. 只有用户明确说“台湾清华大学”“新竹清华大学”或相关表达时，才按台湾高校处理为繁体中文

这个规则优先级高于用户没有特别说明时的默认语言习惯。

## 推荐输入模板

```text
人物肖像图片：已上传
学校名称：必须确认
英文名 / 签名名：
毕业年份：不填时，应届毕业方向默认当前年份
专业 / 学院：例如计算机、法学、建筑
个人特质 / 希望呈现的气质：例如勇敢、自由、坚定
校园经历 / 社团经历：例如学生会、辩论队、实验室项目
```
